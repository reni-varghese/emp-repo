Single Login: Angular to Blazor

How to sign in once in Angular and open the Blazor app without a second login prompt

1) Goal

You want users to login only in the Angular client, then navigate to the Blazor WebAssembly app already authenticated.

Current state in your solution:
  • Angular login returns JWT from API and stores it in localStorage.
  • Blazor has its own login page and also uses JWT from localStorage for auth state.
  • Angular and Blazor run on different ports, so localStorage is not shared automatically.

2) Why token handoff is needed

Browser storage is scoped by origin. Angular origin and Blazor origin are different, so the JWT saved by Angular cannot be read directly by Blazor.

The practical development approach is to hand off the token when navigating from Angular to Blazor.

3) End-to-end flow

  1. User logs in on Angular.
  2. Angular stores JWT in localStorage.
  3. User clicks "Open Blazor App".
  4. Angular redirects to Blazor with token in query string.
  5. Blazor login route reads token, stores it in its own localStorage, and notifies auth state provider.
  6. Blazor redirects to home/employees page as authenticated user.

4.2 Add URL builder in auth service

import { API_BASE_URL, BLAZOR_CLIENT_URL } from '../core/config/client-config';

// Existing service code omitted for brevity

getBlazorLaunchUrl() {
  const token = this.getToken();
  if (!token) return null;

  // URL-safe encoding of JWT
  const encodedToken = encodeURIComponent(token);
  return `${BLAZOR_CLIENT_URL}/login?token=${encodedToken}`;
}

5) Blazor changes

5.1 Convert Blazor login page to token-consumer route

File: EmployeeClient/Pages/Login.razor

@page "/login"

@inject CustomAuthStateProvider AuthStateProvider
@inject NavigationManager Nav
@inject IJSRuntime JS

<h3 class="text-primary">Sign In</h3>

@if (isProcessingToken)
{
    <p>Signing you in and redirecting...</p>
}
else
{
    <div class="alert alert-info col-6">
        Login is managed by the Angular client.
        <a href="http://localhost:4200/login">Go to Angular Login</a>
    </div>
}

@code {
    private bool isProcessingToken = true;

    protected override async Task OnInitializedAsync()
    {
        var token = GetTokenFromQuery();
        if (string.IsNullOrWhiteSpace(token))
        {
            isProcessingToken = false;
            return;
        }

        await JS.InvokeVoidAsync("localStorage.setItem", "token", token);
        AuthStateProvider.NotifyUserLoggedIn(token);
        Nav.NavigateTo("/", true);
    }

    private string? GetTokenFromQuery()
    {
        var uri = Nav.ToAbsoluteUri(Nav.Uri);
        var parsed = Microsoft.AspNetCore.WebUtilities.QueryHelpers.ParseQuery(uri.Query);

        if (!parsed.TryGetValue("token", out var tokenValues))
        {
            return null;
        }

        var token = tokenValues.ToString();
        return string.IsNullOrWhiteSpace(token) ? null : token;
    }
}

6) Security considerations

Important: Passing JWT in query string is acceptable for local development and demos, but not recommended for production because URLs may be logged in browser history, reverse proxies, and server logs.

Production-safe approach (recommended)

Use a short-lived one-time handoff code, not the raw JWT in URL:

  1. Angular calls API with bearer token and requests one-time code (valid ~30-60 seconds).
  2. API stores mapping code to token (or user session) in cache.
  3. Angular redirects to Blazor with ?code=abc123.
  4. Blazor sends code to API exchange endpoint.
  5. API returns JWT (or refresh/access set) and invalidates the code.

8) Troubleshooting

  • 401 in Blazor API calls: verify token exists in Blazor origin localStorage, not only Angular origin.
  • Claims/roles missing: confirm JWT role claim name used by your auth provider.
  • Redirect loop: ensure token query is consumed once and then navigate to root.
  • CORS issue: confirm API CORS allows both Angular and Blazor origins.

9) Optional hardening

  • Always use HTTPS in all environments.
  • Set short access token expiration and implement refresh flow.
  • Add server-side token introspection/blacklist for logout-all scenarios.
